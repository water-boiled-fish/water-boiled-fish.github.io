Github是什么，有什么用

众所周知，Github和推特，油管，脸书一样是一个小公司开的网站，因为它们时不时就会无法访问(由于没钱所以服务器太烂了emmmm不开玩笑了，Github是一个可以托管代码的外国网站，国内也可以直连，就是有时候网速有点慢，如果只是能托管代码的话，为什么不直接存到电脑上呢？因为在Github上你可以发现别人做的项目，复制(克隆)到你的仓库里面，而且可以看到你喜欢的项目，在上面你也可以自己写出项目，和其他人一起交流经♂验。(反正一些摸鱼的项目是真的棒)

开始使用Github

首先我们需要注册一个Github的账号，注册的时候使用qq邮箱激活的话可能会收不到右键，这时候需要在你的qq邮箱里面设置域名白名单，加入github的域名，然后就可以收到邮件了。(收到邮件就不用我多说什么了吧，毕竟干这方面的大部分都是lsp了，懂的都懂)注册成功之后我们就可以搭建属于自己的hexo博客了(幻视)，咳咳跑题了，赶快扯回来，反正Github是一个资源干货很丰富的平台，我相信学会使用git和Github的使用方法之后会更好的提升自己的代码能力(绝对不是因为这个好玩)

仓库

仓库

仓库是一个你可以存放代码的地方，当然如果是新注册的账号的话肯定是没有仓库的，这时候需要你新建一个仓库，只有新建了仓库才可以进行代码的存放，只要不停下来，仓库就会不断延伸，所以说，不要停下来啊。新建仓库的时候你可以自定义你仓库的名字，还可以手动选择你仓库的公开方式，但是请注意,私密的仓库是要收费的（私密的仓库可以设置仅自己可以复制仓库里面的文件或者选择特定用户分享自己的仓库文件），所以说只要不是私密性的开发项目，建议还是选择白嫖的仓库来的更香，毕竟开源的仓库可以促进社区更好的发展。

在别人的仓库中添加自己的内容

这就是对仓库的基础修改，我在上面说过，Github上面是有一部分的社交性的，因为你可以在Github上和仓库的持有者进行互动，合作项目，帮助开发者反映问题，接下来展示一下基础的提交文件的功能

刚进实验室的时候要在Github上交课表，无奈当时没有学会Github的用法，所以我现在想教教后来者如何办成这件事，并记录一下我的学习成果

假设一个情景，现在有人在Github上创建了一个开源项目，你想使用这个项目，于是你把这个项目的仓库克隆了一份，并且开始使用，使用了一小会儿你发现这个程序还有改进的空间，这时候你可以修改(你)自己仓库的内容，把这个项目优化成你想要的样子，并上传至Github上的仓库，这时候你想让该项目的开发者看到你的改动，你可以提交一个合并仓库的申请，等到开发者同意之后你修改的项目就同步到了源项目的仓库中了，这有益于开源社区的发展(梅开二度)具体流程如图，现在就让我来讲一下这个过程是如何实现的ヾ(=･ω･=)o
